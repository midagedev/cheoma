# 토담 표면 표현 계약

> - **상태**: 구현 계약 + 리서치
> - **관련 이슈**: GitHub #135, 상위 폴리시 #75
> - **조사 기준일**: 2026-07-24
> - **대상**: 일반 마을의 `mud` 담장 표면과 그 거리·우천 변화
> - **비대상**: 특정 마을·연대·장인의 배합과 공정에 대한 실측 복원

이 문서는 단색 상자에 가까운 토담을 재료와 축조 방식이 읽히는 표면으로 개선하되, 공개 자료가
확정하지 않는 흔적을 역사적 표준처럼 만들지 않기 위한 경계다. **한국 전통 담장에 관한 직접
근거**, **토벽의 일반 물성**, **cheoma의 제품 해석**을 분리한다. 원문·사진·도판은 저장소나
앱 번들에 복제하지 않으며, 사용자에게 공개할 출처와 이용조건은 [`credits.md`](credits.md)의
42·43번 항목에도 같은 범위로 남긴다.

## 1. 근거가 확인하는 범위

| 표현 후보 | 확인된 사실 | 근거 강도 | 확인되지 않은 것 |
| --- | --- | --- | --- |
| 흙 몸체와 짚 | 토담은 진흙·지푸라기·석회에 잔돌을 섞어 쌓은 농촌 담장으로 설명된다. | 한국 담장 직접 근거 | 배합비, 짚 길이·노출 밀도, 완성면 색 |
| 판담 널틀 | 하회 판담은 널빤지 틀 안에 진흙·돌·지푸라기·석회를 넣어 굳힌 뒤 틀을 뗀다. | 특정 지역 직접 근거 | 전국 보편성, 판 크기, 표면에 남은 이음의 형태 |
| 층상 다짐 | 판축은 판틀 안에 흙이나 모래를 층상으로 넣고 다져 차례로 올리는 공법이다. | 공법 일반 근거 | 이 설명을 조선 민가 판담의 보편 공정·가시적 줄무늬로 등치할 근거 |
| 낮은 돌 하부 | 산청 단계마을 토석담은 하부에 큰 돌을 쌓고 그 위에 작은 돌과 진흙을 교대로 올렸다. | 특정 마을 직접 근거 | 모든 토담의 석재 굽, 60–90cm라는 사례 치수의 전국 적용 |
| 하부 수분 영향 | 토벽 보존 문헌은 비의 습윤·건조, 모세관 상승, 벽 아래의 빗물 튐이 침식과 관련됨을 정리한다. | 국제 토벽 물성 근거 | 한국 토담의 역사적 얼룩 색·높이·계절별 평균 |

### 1.1 흙·지푸라기·잔돌

한국학중앙연구원 「담」은 토담을 진흙·지푸라기·석회에 잔돌을 섞어 쌓은 담으로 설명하고,
하회 판담은 널틀 안에 진흙·돌·지푸라기·석회를 넣어 굳힌 뒤 탈형한다고 구체화한다. 따라서
흙 몸체 안에 **드물고 짧은 식물 섬유**와 소수의 작은 골재를 암시할 수 있다.

다만 출처는 완성 표면에 짚이 얼마나 노출되는지, 섬유가 어느 방향으로 놓이는지, 석회가 색을
얼마나 밝히는지 정하지 않는다. 화면 전체를 덮는 긴 노란 선, 볏짚 다발처럼 튀어나온 섬유,
균일한 백색 회벽은 근거를 넘는다. 짚은 가까운 거리에서 재료를 식별시키는 낮은 빈도의 보조
어휘이며 토담의 주 실루엣이 아니다.

### 1.2 판담과 판축은 같은 주장으로 합치지 않는다

국립문화유산연구원 「판축」은 판을 양쪽에 대고 흙·모래를 층상으로 넣어 다져 올리는 일반
공법을 설명한다. 이는 판틀과 단계적 다짐이 수평 리듬을 만들 수 있다는 **공정상의 가능성**을
제공한다. 그러나 이 자료의 주요 한국 사례는 성곽·기단·고분이며, 하회 판담의 보편 lift 높이나
완성면에 남은 자국을 측정한 자료가 아니다.

제품은 이 차이를 다음처럼 보존한다.

- 낮은 대비의 넓고 불규칙한 수평 band로 `차례로 올린 흙`을 암시할 수 있다.
- 소수의 세로 이음 또는 완만한 면 차이는 널틀을 읽히게 하는 제품 해석이다.
- 일정 간격의 벽돌 줄눈, 깊은 홈, 모든 run에서 정확히 이어지는 장식 띠는 금지한다.
- band 간격·이음 위치·깊이를 역사적 치수나 전국 표준으로 표시하지 않는다.

### 1.3 돌 하부는 가능한 어휘이지 전국 규격이 아니다

국가유산청의 산청 단계마을 옛 담장 설명은 그 마을 토석담의 하부에 큰 돌을 60–90cm 정도
쌓고, 그 위에 작은 돌과 진흙을 교대로 쌓았다고 기록한다. 이는 흙 몸체를 바로 지면에 꽂은
단일 상자보다 돌 하부가 가능한 전통 담장 어휘임을 확인한다.

현재 `mud` 담장은 이미 낮은 `fieldstone` 굽을 구조적으로 갖는다. 이 굽은 그대로 footprint와
충돌 계약을 소유하며, 단계마을의 60–90cm를 그대로 축소 복제한 실측물이 아니다. 새 표면
효과는 이 굽을 가리거나 별도의 충돌체·두께·높이를 만들지 않는다. 돌 하부를 모든 시대·지역
토담의 필수 형식으로 문서화하지도 않는다.

### 1.4 습윤은 일반 물성에서 번역한다

Getty Conservation Institute의 *Terra Literature Review*는 토벽 열화에서 비에 의한
습윤·건조, 모세관 상승, 벽 하부의 splash-back을 구분한다. 이 자료는 한국 담장 형식의
역사자료가 아니라 토질 건축 보존 연구를 정리한 국제 문헌이다.

따라서 지면 가까운 낮고 불규칙한 습윤 영역은 물리적으로 타당한 **제품 번역**으로만 쓴다.
정확한 색과 높이는 실측값이 아니며 다음을 지킨다.

- 비가 오면 아래에서 시작하는 저주파 darkening이 연속적으로 강해지고, 그치면 천천히 약해진다.
- 마른 상태에는 매우 약한 누적 흔적만 허용하며 검은 띠처럼 둘레를 균일하게 감지 않는다.
- 기본 표현에 녹색 이끼, 검은 곰팡이, 염류 백화, 깊은 침식 구멍을 함께 발명하지 않는다.
- 습윤은 roughness·색·미세 relief의 절제된 변화이며 발광, 투명도, `polygonOffset`이 아니다.
- 지면과 만나는 돌 굽·담 몸체의 경계는 유지하고, 비 표현이 그 구조를 지우지 않는다.

## 2. cheoma 제품 계약

### 2.1 순수 profile이 의미를 소유한다

토담 표면의 seed와 구조 결정은 Three.js·DOM·전역 `Math.random`에 의존하지 않는 순수
profile에서 만든다. renderer가 mesh 순회 순서, 월드 좌표의 우연한 소수점, 현재 카메라로
다시 무늬를 뽑아서는 안 된다.

profile은 최소한 다음 의미를 한 번만 결정한다.

- run을 식별하는 seed-stable surface identity
- 제한된 수의 넓은 수평 band와 선택 세로 이음
- 드문 짚 섬유·작은 골재의 위치, 방향, 크기
- 돌 굽 위에서 시작하는 정규화된 습윤 범위와 저주파 변주
- `lift`, `fibre`, `damp` 기여를 독립적으로 끌 수 있는 A/B 축

값은 renderer 단위가 아니라 run 로컬의 정규화 좌표와 의미 record로 노출한다. 순수 공개
entrypoint는 JSON 직렬화 가능한 불변 결과만 반환하고 `THREE.Material`, shader, geometry를
노출하지 않는다. 같은 입력·seed는 sync, Worker, fallback과 외부 프로젝트에서 같은 결과를
내야 한다.

### 2.2 구조 계약을 바꾸지 않는다

표면 개선은 [`village-walls-parcels.md`](village-walls-parcels.md)와
`src/village/wall-contract.js`가 소유한 배치 계약의 소비자다.

- `mud`의 polygon, 중심선, 0.5m 솔리드 두께, 높이, 수평 단, gate split·landing을 바꾸지 않는다.
- 기존 낮은 `fieldstone` 굽과 몸채에 맞춘 기와/이엉 coping의 높이·실루엣을 보존한다.
- corner post와 run grow가 가리는 구조 이음에 표면 자국을 억지로 관통시키지 않는다.
- picking, focus blocker, 일조, yard clearance는 기존 구조 record만 계속 사용한다.
- 표면 relief가 필요하면 기존 몸체 envelope 안의 미세 변화로 제한하고 새 planning footprint가
  되지 않게 한다.

### 2.3 거리·날씨·모드

- **근경**: 얕은 다짐 리듬, 드문 섬유, 낮은 습윤 변화가 실제 조명과 깊이를 받아 읽힌다.
- **중경**: 섬유·작은 골재는 사라지고 band와 하부 색조만 낮은 대비로 합쳐진다.
- **원경**: 기존 토담 실루엣과 평균색만 남고 추가 detail의 simulation과 draw를 잠재운다.
- 계절은 흙 배합을 다시 뽑지 않는다. 날씨는 같은 profile의 습윤 weight만 tween하며 hard cut이나
  별도 젖은 mesh를 만들지 않는다.
- PBR color와 shadow/depth/DoF가 서로 다른 표면 위치·coverage를 쓰지 않는다. 수묵 모드에서는
  섬유 선을 과장하지 않고 큰 면·돌 굽·수평 리듬만 보존한다.
- exclusive wave 중에는 기존 벽 subtree의 소유권과 가시성 전환을 따르며 별도 crossfade material을
  만들지 않는다.

### 2.4 성능·수명주기

- 담장 수에 비례하는 material clone, texture, shader program, draw call을 만들지 않는다.
- 기존 공유 재질과 정적 병합 경계를 유지하고, 가까운 tier의 물리 detail도 bounded batch로 제한한다.
- profile 생성은 run당 고정 상한을 가지며 후보 수가 길이에 따라 무제한 늘지 않는다.
- shader를 patch한다면 한 번만 설치하고 모든 mud run이 같은 program을 공유한다.
- rebuild·reroll은 새 profile과 render 자원을 원자적으로 교체하고, 실패하면 기존 담장을 보존한다.
- dispose는 자신이 만든 geometry·attribute만 해제하고 palette가 빌려준 material을 소유하지 않는다.

## 3. 시각·회귀 검증

같은 seed와 카메라에서 다음 비교를 남긴다.

1. 기존 / 전체 개선: 실제 제품의 주거 근경과 마을 부감.
2. `lift` off / on: 규칙적 벽돌 줄눈이나 장식 띠처럼 읽히지 않는지.
3. `fibre` off / on: 짚이 표면 전체의 노란 털이나 낟가리로 변하지 않는지.
4. `damp` off / on: 하부가 검은 균일 띠가 아니며 돌 굽과 지면 접촉이 읽히는지.
5. clear → rain → clear: 같은 무늬가 연속적으로 젖고 마르며 hard cut·seed 변화가 없는지.
6. FULL → MID → FAR: detail이 튀거나 크기가 역전되지 않고 평균색으로 자연스럽게 축약되는지.
7. PBR / ink / DoF / snow / wave: coverage·depth·그림자·소유권이 서로 어긋나지 않는지.

정량 기록은 sync/Worker/fallback plan hash, program·draw·triangle·texture delta, 최대 profile
record 수를 포함한다. headless 절대 frame-ms는 채택 근거로 쓰지 않는다. 최종 시각 검수에서는
실제 Chrome에서 집과 대문이 주인공으로 남고 담의 대비가 건축을 압도하지 않는지도 확인한다.

# 한양 시전행랑 표현 계약

> - **상태**: 조사·구현 계약
> - **관련 이슈**: GitHub #128, 상위 폴리시 #75
> - **조사 기준일**: 2026-07-24
> - **대상**: 한양 중심 간선변의 시전행랑
> - **비대상**: 특정 연도·발굴지·개별 시전의 실측 복원

이 문서는 시전행랑을 단순한 흰 상자 열에서 벗어나게 하되, 사료가 말하지 않는 점포 입면을
사실처럼 만들지 않기 위한 경계다. **확인된 사실**, **남아 있는 한계**, **cheoma의 경량 제품
해석**을 분리한다. 원문 문장·사진·도면·발굴 이미지는 저장소나 앱 번들에 복제하지 않는다.
사용자에게 공개할 선별 출처와 이용조건은 [`credits.md`](credits.md)의 40번 항목에도 같은 범위로
남긴다.

## 1. 출처가 확인하는 사실

### 1.1 가로를 따라 이어진 행랑과 칸

국사편찬위원회 우리역사넷 「조선 전기 신도의 시전 상업」은 1412년(태종 12)부터 세 차례의
공사로 한양의 행랑 2,027칸이 조성되었고, 운종가에서 종묘 앞 누문 사이와 종루에서 광통교
사이 등이 시전 전용 행랑으로 쓰였다고 설명한다. 2,027칸 전체가 모두 시전이었다는 뜻은
아니다.

서울역사박물관 「청진동 발굴 유물」은 청진지구에서 **종로를 따라 일렬로 늘어선 시전행랑**,
그 뒤의 피맛길, 주택가와 막다른 골목이 함께 확인되었다고 설명한다. 따라서 제품이 가져올
핵심 공간 언어는 독립 점포 상자의 반복이 아니라 다음과 같다.

- 간선에 평행한 긴 가로벽과 처마선
- 그 선을 나누어 읽게 하는 반복 칸과 기둥
- 전면 시전행랑 뒤에 이어지는 피맛길·배후 공간의 방향성

이 근거는 모든 시전이 한 동의 끊김 없는 지붕이었다거나, 각 칸의 실측 폭·깊이가 같았다는
뜻은 아니다. cheoma의 기존 점포 footprint와 pitch는 도시 배치·가독성을 위한 제품 수치로
남는다.

### 1.2 칸은 사용과 관리의 단위였다

같은 우리역사넷 자료는 초기 시전 상인이 행랑 **한 칸마다** 봄·가을 공랑세를 냈다고
기록한다. 한국사 연대기 「금난전권」은 한성부가 시전의 행랑 규모를 칸 수, 상인 이름,
수공업자와 함께 시안에 기록해 관리했다고 설명한다. 「도가와 시전 행랑」은 조선 후기에
낡거나 불탄 행랑을 민간 자본으로 다시 짓고 시전 상인에게 임대료를 받았다고 설명한다.

그러므로 칸별 반복을 영업 단위로 읽히게 하는 것은 근거가 있다. 다만 후기 임대료의 정확한
산정 단위·금액까지 칸별이었다고 단정하지 않는다. 제품에서는 하나의 계획 칸을 하나의
가벼운 점포 의미 단위로 사용하되, 역사적 세율이나 소유관계를 시뮬레이션하지 않는다.

### 1.3 표식과 물종 어휘

우리역사넷은 초기 시전행랑에 표를 세워 **시전의 이름과 판매 물종**을 표시했다고 설명한다.
한국사 연대기도 시명과 판매 물종이 기록되었다고 교차 확인한다. 이는 현대식 개인 상호,
대형 fascia 간판, 네온 광고의 근거가 아니다.

「유분각전과 무분각전」은 비단·무명·종이·모시·생선·곡물·유기·신발·철물 등 시전별
판매 물종의 넓은 어휘를 제공한다. 제품은 이 목록을 다음처럼 거친 실루엣 군으로만 축약한다.

| 제품 물종 군 | 가까이서 읽힐 최소 어휘 | 배제 |
| --- | --- | --- |
| 직물·종이 | 접거나 만 얇은 묶음 | 읽을 수 있는 현대 포장·인쇄 상표 |
| 곡물·마른 물건 | 낮은 포대·바구니 | 실제 식품 사진 텍스처 |
| 어물 | 절제된 건어물 묶음·바구니 | 젖은 생선의 사실적 대량 진열 |
| 유기·철물 | 소수의 그릇·기물 실루엣 | 고광택 금속 과장·상품 과밀 |
| 신발·생활 잡화 | 작은 쌍·상자 | 시대가 다른 고무신·현대 진열대 |

시대가 지정되지 않은 기본 장면에는 도입 시기가 분명히 늦은 담배 같은 물종을 대표 표식으로
강조하지 않는다. 작은 표식은 `시명/물종이 있었다`는 사실만 전달하며, 정확한 판 모양·서체·색·
부착 높이는 제품 결정으로 표시한다.

### 1.4 발굴 유구가 말하는 범위

서울역사박물관은 청진지구 발굴에서 화재와 전란으로 불탄 채 묻힌 기둥, 마룻장, 문짝과
생활 도구가 확인되었다고 설명한다. 박물관 보도자료와 야외전시 안내는 2004년 종로1가·청진동
일대에서 발견된 조선시대 시장 점포 터 일부를 이전·복원했다고 기록한다.

이 자료는 다음 표현을 지지한다.

- 기둥과 목재 수평재가 만드는 칸 리듬
- 바닥 높이와 후퇴한 실내의 얕은 깊이
- 완전히 비어 있는 가설 천막으로 환원할 수 없는 문·마루를 갖춘 목조건축
- 종로 전면, 피맛길, 배후 주거가 층을 이루는 도시 조직

그러나 공개 해설만으로 발굴된 각 문짝이 어느 점포의 어느 전면에 달렸는지, 모든 칸이 같은
방식으로 열렸는지, 완전한 입면과 지붕 결구가 어떠했는지는 복원할 수 없다. 발굴 유구는
실제 건축의 존재와 재료·도시 맥락을 확인하는 근거이지, 임의의 완전 입면을 실측 복원으로
격상시키는 근거가 아니다.

### 1.5 도가와 판매 행랑은 같은 것이 아니다

「도가와 시전 행랑」은 도가를 시전 도중의 사무실·회의 장소이자 창고로, 상품 거래가
이루어지는 시전행랑과 구분한다. 주요 도가의 층수와 규모를 짐작하게 하는 단편 기록은 있지만
구체적인 모습은 거의 남지 않았다고 출처 자체가 한계를 밝힌다.

이번 범위는 도가를 모든 점포 뒤에 반복하지 않는다. 얕은 후면 저장 공간은 판매 행랑의
깊이를 읽게 하는 제품 표현일 뿐, 각 칸을 독립 도가로 재현한 것이 아니다.

## 2. 확인되지 않은 입면과 절제 원칙

다음은 현재 선별 출처가 확정하지 않는다.

- 모든 점포 전면의 개방 비율과 여닫는 방식
- 판매대·상·문턱의 보편 형식, 치수와 위치
- 칸의 보편 실측 폭·깊이·기둥 높이
- 표의 정확한 재료·형태·서체·색·부착 위치
- 지붕이 몇 칸마다 끊겼는지와 모든 구간의 동일한 지붕 형식
- 백색 회벽이 모든 시대·시전에 공통이었다는 주장
- 한 시점에 진열된 상품의 양과 점포별 분포

특히 우리역사넷이 설명하는 `상전(床廛)`은 물건을 상 위에 놓아 판 특정 시전 어휘다. 이를
모든 시전 전면에 동일한 고정 카운터가 있었다는 증거로 일반화하지 않는다. cheoma가 두는
낮은 bench는 두 칸의 깊이를 반복해서 읽히게 하는 **제품 해석**이며, 보편적인 역사 형식이나
실측 치수가 아니다. 계획 footprint 안에 머물고 보행 회랑으로 내밀지 않는다.

또한 조선 전기 조성과 후기의 화재·재건·임대 기록, 여러 시기의 발굴 층위를 한 날짜의 모습으로
합치지 않는다. 현재 장면은 시전행랑이라는 장기 지속 공간 문법을 표현하는 것이며 특정 연도의
종로 복원도가 아니다.

## 3. cheoma의 제품 번역

### 3.1 순수 계획이 소유할 것

`src/village/sijeon-plan.js`는 Three.js와 전역 `Math.random`에 의존하지 않고, 기존
`shop.poly`, `center`, `frontDir`, `w`, `d`를 보존한 채 칸별 façade 의미만 파생한다.
renderer가 shop ID나 역할 이름으로 두 번째 배치를 추론해서는 안 된다.

계획은 `planSijeon(roadsResult, site, char01)`로 기존 배치 record를 만들고,
`planSijeonFacade(shop)`으로 각 점포의 로컬 façade를 파생한다. façade 좌표는 `+x`가 행랑
진행 방향, `+z`가 도로를 향한 전면이다. 현재 schema v1은 한 점포를 제품상 2칸으로 나누며
다음 물리 의미를 소유한다.

- placement의 안정적인 점포 ID와 façade 부재의 칸 index
- 세 전면 기둥, 두 상인방과 두 후퇴 개구
- 각 칸의 낮은 display bench
- 후면 저장 매스
- 맞배지붕과 전·후·측면 처마 내밀기
- 일반 부재의 도로 쪽 한계와 처마만 허용하는 명시적 예외

시명·물종 표식과 상품 실루엣을 구현할 때도 칸 ID에서 파생된 순수 의미를 먼저 계획에
기록한다. renderer가 자체 난수나 배열 순서로 두 번째 변주를 만들지 않는다. 두 bench는 칸의
깊이를 읽히게 하는 제품 수치·형식이며, 모든 역사적 시전에 같은 고정 판매대가 있었다는 주장이
아니다.

**schema v1 경계:** 현재 `SIJEON_FACADE_SCHEMA_VERSION=1`은 위 물리 façade만 소유하고 표식·
상품 record는 의도적으로 넣지 않는다. 따라서 v1 renderer의 bench는 비어 있어도 유효하며,
표식이나 상품을 임의로 채워서는 안 된다. #128 통합에서 이를 실제로 렌더하기로 결정하면 전역
RNG 없이 순수 plan의 명시적 후속 schema 필드로 먼저 추가하고, 문서·검증·renderer를 함께
갱신한다.

v1은 임의 변주 없이 기존 점포 record만 사용한다. 전역 RNG를 소비하지 않고, 같은 입력은 바이트
단위로 같은 plan을 만든다. 후속 schema가 변주를 갖더라도 점포 ID에서 만든 독립적인 안정 hash를
사용하고, 장식 때문에 기존 시전 위치·footprint, 도로, 성곽, 필지와 worker 계획 결과가
달라져서는 안 된다.

### 3.2 renderer가 표현할 것

전용 시전 renderer는 계획을 그대로 읽어 다음 순서로 깊이를 만든다.

1. 기존 지붕·측벽·후면 저장 매스로 연속 행랑의 큰 실루엣을 보존한다.
2. 전면 벽을 처마 안쪽으로 후퇴시킨다.
3. 세 전면 기둥과 두 상인방으로 제품상의 두 칸을 다시 읽힌다.
4. 두 개구를 기둥선 뒤로 물리고 판문·목재 면과 깊은 음영을 절제해 빈 동굴이나 현대
   쇼윈도를 피한다.
5. 낮은 display bench는 비운 채로도 칸의 전후 깊이를 설명한다.

현대 유리, 네온·emissive 상호, NPC 군중, 천막 숲, 모든 칸의 상품 과밀은 넣지 않는다. 가까운
화면의 생활감은 우선 기둥 뒤의 깊이·그림자와 빈 bench로 만들고, 원경의 정보량은 긴 처마선과
칸 리듬이 맡는다.

표식·상품 어휘는 역사적 근거로 문서와 References UI에는 남기지만 **rendered v1에서는
제외**한다. 공개 해설이 정확한 판·진열 형식을 확정하지 못하는 상황에서 임의 detail을 보태기보다
열주·상인방·후퇴 개구·bench·후면 저장이라는 최소 요소로 형태 문제를 먼저 해결하는 선택이다.
후속 schema가 생길 경우에도 표식은 작고 비발광이어야 하고, 상품은 표의 제품 물종 군을 소수
실루엣으로만 사용해야 한다.

## 4. 성능·표현 경계

### 4.1 병합과 자원

- 점포 수와 무관하게 역할별 geometry를 모아 하나의 `village-sijeon` 정적 병합 그룹으로
  유지한다. 목표는 새 텍스처 0, 약 5 draw call이며 칸마다 Mesh·Material을 만들지 않는다.
- v1의 개구·bench는 공유 palette와 병합된 geometry로 표현한다. 후속 물종·표식도 새 material
  variant를 만들지 않고 vertex color 또는 병합된 geometry 차이만 사용할 수 있다.
- 계획은 숫자·문자열·작은 배열만 반환하고 Three 객체나 dispose 책임을 갖지 않는다.
- renderer가 만든 geometry와 palette clone은 그룹 수명에 맞춰 한 번 해제되며, 호출자 공유
  재질의 소유권을 빼앗지 않는다.

### 4.2 거리와 LOD

- **원경**: 지붕과 연속 가로벽, 큰 칸 간격만 읽혀야 한다. 작은 상품·표식의 색이나 글자를
  원경 가독성 조건으로 삼지 않는다.
- **중경**: 열주, 상인방, 후퇴한 전면의 명암이 흰 상자 실루엣을 깨야 한다.
- **근경**: 기둥선 뒤의 두 개구, 낮은 bench와 후면 저장 매스가 깊이를 설명해야 한다.

별도 FAR impostor나 점포별 `THREE.LOD`를 추가해 두 표현을 동시에 유지하지 않는다. 작은 detail을
거리에서 숨길 필요가 있다면 이미 병합된 의미 묶음을 안정적인 그룹 단위로 제어하고, 화면 크기에
비해 보이지 않는 요소를 처음부터 절제하는 것을 우선한다. 거리 전환 때문에 draw call, material
program 또는 텍스처가 점포 수에 비례해서는 안 된다.

### 4.3 수묵·눈·림라이트

- 수묵은 같은 물리 geometry와 깊이를 소비한다. 기둥·처마·후퇴선은 색 없이도 먹선과 농담으로
  읽혀야 하며, 별도 ink 전용 mesh나 material을 만들지 않는다.
- 기와 지붕만 기존 roof 역할과 `snowSurface` 계약을 통해 눈을 받는다. 수직 열주, 열린 전면,
  bench와 후면 저장 면에 지붕용 적설을 누출하지 않는다. 새 눈 텍스처나 눈 geometry를 추가하지
  않는다.
- 석양 림은 실제 sun-facing silhouette와 shadowed main-light 조건을 통과한 기존 물리 rim
  patch만 받으며, 어두운 처마 아래를 임의로 밝히지 않는다. 미래의 표식·상품도 발광하지 않는다.

### 4.4 리롤 wave

`village-sijeon`은 기존 wave 분류의 **그룹 단위** 구조물이다. 구·신 시전을 투명 재질로
crossfade하지 않고, 먹안개 handoff 아래에서 배타적으로 소유권을 넘긴다. 그룹의 제 바운딩 하단을
pivot으로 기존 tofu scale·drop을 적용하며 다음을 금지한다.

- `transparent`, `opacity`, `depthWrite`를 전환용으로 변경
- 구·신 행랑의 동시 가시와 겹친 그림자
- wave 동안 임시 material/program 생성
- 완료·취소 후 변형 scale, visibility, geometry 소유권 잔류

## 5. 완료 체크리스트

### 역사성과 해석

- [ ] 긴 간선변 행랑, 칸 단위 사용, 시명·물종 표식, 발굴 유구의 범위를 구현 설명과 Reference
      UI에서 확인할 수 있다.
- [ ] 정확한 개방 전면·판매대·칸 치수·표식 모양은 제품 결정임을 명시한다.
- [ ] 도가를 모든 판매 칸의 후면으로 오해하지 않는다.
- [ ] 특정 연도의 종로 또는 청진동 발굴지 실측 복원이라고 표시하지 않는다.

### 계획과 배치

- [ ] 기존 시전 위치·footprint·도로·성곽·필지 결과가 그대로다.
- [ ] 전역 `Math.random` 소비가 0이고 동일 입력의 plan과 geometry 결과가 동일하다.
- [ ] 열주·bench·후면 저장 매스와 처마의 명시적 예외가 도로 회랑과 인접 footprint 계약을
      지킨다.
- [ ] renderer가 칸·물종·개방 상태를 다시 추론하지 않고 순수 plan을 소비한다.

### 시각과 성능

- [ ] 낮·석양·수묵에서 거리 아이레벨, 사선 부감, 한양 전체 부감을 직접 비교한다.
- [ ] 원경은 연속 처마선, 중경은 칸 리듬, 근경은 후퇴 깊이와 절제된 생활감이 읽힌다.
- [ ] 새 텍스처 0, 점포 수와 무관한 단일 병합 그룹, 약 5 draw call 목표를 지킨다.
- [ ] snow·물리 rim·ink·wave와 dispose 계약을 자동 검사한다.
- [ ] worker scene hash 변화는 의도한 Hanyang 시전 geometry에 한정되고 다른 규모와 picking
      계약은 유지된다.

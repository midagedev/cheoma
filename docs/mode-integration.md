# 모드 통합 기획 — "하나의 월드, 하나의 포커스" (2026-07-18)

사용자 피드백: 마을편집/집편집/히어로 모드의 통합이 매끄럽지 않다. 기획부터 재구성.
요구: ① 인상적이면서 통합될 것 ② 줌인 시 앰비언스 자산(소동물·파티클 등)이 살아날 것 ③ 궁·종가 등 특수 건물도 편집 가능할 것.

## 1. 문제 진단 (왜 삐걱였나)

지금까지 "집 모드"(단독 씬, env 풀 앰비언스)와 "마을 모드"(adapter 씬, 제한 앰비언스)가 **별개 세계**로 존재했고,
히어로/편집/리플레이가 그 사이를 오가는 **전환**으로 구현됐다. 전환마다 씬·조명·앰비언스가 갈아끼워지니
이음새가 드러나고, 마을에서 줌인하면 집 모드에서 만든 생명감(닭·연기·모트·등롱 흔들림)이 없다.

## 2. 새 모델: 모드가 아니라 포커스

**월드는 항상 마을 하나다.** 사용자에게 보이는 상태는 카메라와 포커스뿐:

```
state = { focus: null | parcelId }
```

- `focus == null` → **부감(마을 배율)**: 마을 전체, 마을 파라미터 패널(규모·시드·지형·리롤 웨이브).
- `focus == parcelId` → **근접(집 배율)**: 그 필지에 카메라 돌리인, 풀디테일 오버레이, 집 파라미터 패널.
- **히어로 랜딩 = 부팅 연출**: 마을 생성 → 타이틀 → 종가로 자동 focus-in + 조립 애니. 별개 모드가 아니라
  "focus 전환의 연출 버전". 리플레이 버튼 = 같은 연출 재실행.
- 기존 "집 단독 모드"는 사용자 개념에서 제거한다 (URL/샷 하네스 경로는 내부 호환 유지).

### 전환 규약
- focus-in: 필지 클릭(모바일 탭) → 카메라 돌리인(매 프레임 lookAt, 종료 관성 리셋 — camera-tween 계약)
  → 도착 시 패널 슬라이드 인. 돌리 중 DoF 램프(기존 village 돌리인 자산 재사용).
- focus-out: 뒤로/배경 액션 → 돌리아웃 → 마을 패널 복귀.
- 모든 전환에 env 크로스페이드(#50)·present-gate(#61) 적용 — 팟·조기노출 금지.

## 3. 앰비언스 근접 링 (신규 핵심 시스템)

**원칙: 앰비언스는 배율에 종속.** 부감은 저비용 세트(원경 창불 #60, 구름 그림자, 운해, 굴뚝 연기 몇 가닥),
focus-in 하면 그 필지 반경에 **풀 앰비언스 링**이 점등된다:

| 자산 | 부감 | focus-in (근접 링) |
|---|---|---|
| 소동물 (마당 닭, 논 소) | off | on — setupAnimals 재사용 (env/animals.js 재사용 API 기존재) |
| 굴뚝 연기·아궁이 (smoke.js) | 원경 몇 가닥 | 해당 채 풀 디테일 (building 조회 배선 기존재) |
| 먼지 모트 (motes.js) | off | on — 마당 볼륨 |
| 등롱 바람 흔들림 | off | on |
| 창호 글로우·석등 | vnight 경로 (기존) | 동일 + 근접 보정 |
| 적설 볼륨·빗물 흐름 (#52) | off (지붕 틴트만) | on — 오버레이 지붕 대상 |
| 물글린트·계절 | adapter 기존 경로 | 동일 |

### 인터페이스 계약 (env ↔ adapter/app)
```js
// src/env/focus.js (신규)
const ring = createFocusRing(scene, { heightAt });
ring.set({ group, parcel, radius = 18, seed });  // group=풀디테일 오버레이(트래버스로 굴뚝·마당·지붕 앵커 탐지)
                                                  // parcel={ yard, cowSite?, bounds } (populate 필지 데이터)
ring.clear();                                     // focus-out — 페이드 아웃 후 dispose
ring.update(dt, timeName);                        // 매 프레임
```
- 활성화·해제 모두 크로스페이드, present-gate 게이트. 링은 **동시 1개만** (드로우콜 규율).
- env 검증은 /__amb 하네스 문법(앱 경로 독립)으로.

## 4. 편집 — 특수 건물 포함 전 필지

- **모든 필지가 편집 대상이다. 궁·절·종가 포함.** focus-in 하면 편집 가능 — 예외 없음.
- 편집 반영 경로: focus 필지는 풀디테일 오버레이(buildParcel)이므로 파라미터 변경 = 오버레이 재생성.
  - hanok(종가 포함): `buildParcel({ roofOpts, wallH })` 포워딩 (반영 완료).
  - palace/temple/choga: `buildParcel({ presetOverrides })` → `buildBuilding({...PRESETS[preset], ...overrides})` 병합 (반영 완료).
- 파라미터 축은 유형별 프리셋: 기와집(지붕 물매·처마·비례·외피·창호), 궁·절(+단청 강도·공포·월대), 초가(지붕 두께·마당).
- **파라미터 전수 노출 원칙 (사용자 지적 2026-07-18)**: #55 대확장(칸 너비·수직 비례·외피·창호·마당 부속)·#37 담장
  어휘 등 코어가 이미 지원하는 축이 편집 패널에 하나도 안 나오고 있다. **코어가 지원하는 축은 패널에 전부 노출한다**
  — 유형별 그룹핑 + 기본/고급 접기로 과밀만 제어. 마을 패널(부감)도 동일 원칙(규모·시드·지형 기복·담장 스타일·
  다양성 강도 등 plan/populate 옵션 노출). 패널은 하드코딩이 아니라 **파라미터 스키마 주도(schema-driven)** 로
  구성해 코어 축 추가가 자동 반영되게 한다.
- #69 라이브 반영(드래그 중 실시간)은 focus-in 패널에서 완성한다 — 절차 생성 데모의 심장.
- 임포스터(far 청크) 필지는 focus-in 시 풀디테일 오버레이로 승격되므로 편집 문제 없음 (#47 한계 해소 경로).

## 5.5 v2 — "하나의 패널, 하나의 줌 연속체" (사용자 재피드백 2026-07-18)

사용자: "아직도 마을편집/집편집/히어로가 스무스하게 통합되지 않는다 — 목적에 맞게 멋있게." v1(focus 모델)은
상태·기능 통합이었고, 남은 이음새는 **연출과 UI 표면**이다. 진단:

- (a) 패널이 여전히 두 개(VillagePanel ↔ VillageEditPanel)로 **갈아끼워져** "두 모드" 인상 잔존.
- (b) 遠/近가 **이산 토글** — 중간이 없는 순간 점프라 "하나의 공간"으로 체감되지 않음.
- (c) 히어로 랜딩·리플레이가 별도 경로로 읽힘 (focus-in 의 연출 버전이라는 기획이 UI 로 드러나지 않음).
- (d) 전환 시 카메라·패널·앰비언스 링·DoF 가 각자 타이밍으로 움직여 리듬 불일치.

### v2 원칙
1. **줌 연속체**: 휠/핀치로 부감↔근접이 연속. 줌 인 중 카메라가 필지에 충분히 가까워지면(중심 픽킹) 그 필지가
   자동 focus — 스냅 돌리로 마무리. 遠/近 토글은 이 연속체의 숏컷 버튼일 뿐(유지하되 개념은 줌).
2. **단일 컨텍스트 패널**: 패널은 하나. 헤더에 컨텍스트 브레드크럼(마을 → 필지). 부감=마을 섹션,
   근접=그 집 섹션이 **같은 패널 안에서 모프/슬라이드** — 패널 교체 금지. 스키마 주도 렌더러는 이미 있으니
   스키마 소스만 컨텍스트로 스위칭.
3. **전환 타임라인 통일**: focus-in = 돌리인 + DoF 램프 + 링 크로스페이드 + 패널 컨텍스트 모프를
   **한 타임라인**(동일 duration·이징)으로 구동. focus-out 은 역재생. 개별 시스템이 각자 트윈하지 않는다.
4. **히어로 = 첫 focus-in, 리플레이 = 현재 필지 재조립**: 리플레이(再)를 종가 한정에서 **모든 focus 필지**로
   일반화(오버레이 조립은 이미 필지 무관). 부팅 랜딩과 리플레이가 같은 코드 경로가 되도록.
5. **마을 패널 = 마을의 편집기** (사용자: "마을 패널도 꽤 디테일하게 줄 수 있는 게 많을 것"):
   규모 슬라이더 외에 코어 옵션 전수 노출 —
   지형(기복 강도 undAmp·배산 높이·개울), 구성(논 밀도·나무 밀도·궁/절·시전·성곽[도성급 자동 게이트]),
   어휘(담장 스타일 분포·초가/기와 비율·다양성 강도 variants), 그리고 "다시 짓기"(=#56 리롤 웨이브) 버튼.
   → plan/populate 옵션 표면화(코어)가 선행 조건. 집 패널과 같은 스키마 문법으로.

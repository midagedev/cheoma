# 건축 고증 감사와 구현 경계

> - **상태**: 계약 + 리서치
> - **기준일**: 2026-07-22
> - **범위**: 궁·사찰·기와집·초가의 지붕 장식, 창호, 부엌·구들, 담장·마당 생활상
> - **목적**: 사진을 닮게 만드는 데서 끝내지 않고, 출처가 말하는 사실과 cheoma의 경량 표현을 구분해 재현한다.

## 1. 감사 방법

고증 판단은 다음 순서로 한다.

1. 국가유산청·국립문화유산연구원·건축공간연구원(AURI) 같은 공공기관의 조사·해설을 우선한다.
2. 한국학중앙연구원 백과와 국사편찬위원회 우리역사넷으로 용도와 생활 방식을 교차 확인한다.
3. 특정 건물의 사진은 그 건물의 사례이지 조선 건축 전체의 규칙으로 일반화하지 않는다.
4. 출처가 확인한 사실, 이 프로젝트가 선택한 번역, 아직 모델링하지 않은 한계를 각각 기록한다.
5. 같은 seed·카메라의 전후 PNG를 직접 열어 실물 사진·도해와 나란히 보고, 수치 게이트는 그 판단을 보조한다.

이 문서는 원문 이미지나 도면을 재배포하지 않는다. 조사 중 내려받은 이미지는 일시적인 비교 자료일 뿐 저장소 자산이 아니다. 사용자가 볼 가치가 있는 선별 출처는 [`credits.md`](credits.md)에 기관, 문서명, 적용 위치, 원문 URL, 이용조건을 함께 싣고 실제 Reference UI에서 확인한다.

## 2. 출처가 확인하는 사실과 구현 해석

### 2.1 살림집 부엌·구들

- AURI 국가한옥센터의 한옥 구성 해설은 안채에 안방·부엌·광을 두고 부엌을 조리 공간이자 온돌의 열원으로 설명한다. 온돌 도해는 부엌의 아궁이, 방바닥 아래 구들, 반대편 굴뚝으로 이어지는 계통을 보여 준다.
- 국사편찬위원회 우리역사넷의 조선시대 부엌 해설은 부엌 바닥이 방보다 낮고 마당과 가까우며, 가마솥이 부뚜막에 걸리고 아궁이에 불을 땠다고 설명한다.
- 한국학중앙연구원의 「구들」·「굴뚝」 항목은 아궁이→고래→굴뚝의 분리된 연기 길을 교차 확인한다.

**구현 해석**: `src/builder/kitchen-hearth.js`는 아궁이와 솥을 집 외벽에 붙인 독립 노천 화덕이 아니라 마당 높이 부엌 개구 안으로 후퇴시킨다. 초가와 기와집이 같은 조립기를 사용하고, 별도의 굴뚝과 기존 연기·야간 불빛 계약은 유지한다. 실제 내부 고래를 만들지 않고 개구의 암부와 제한적으로 보이는 솥으로 공간 관계만 전달한다.

### 2.2 살림집 창호와 장식 위계

- AURI 해설은 살림집 전후면의 창·문, 머름, 살문의 목재·한지 구성을 설명한다. 같은 자료는 일반 살림집에 다듬은 석재, 화려한 공포, 단청 사용이 제약되었고 대체로 소박했다고 정리한다.
- AURI가 공개한 하회·영남 살림집 사례 사진에서는 백골 목재와 중립적인 한지 창호가 읽힌다. 이는 특정 살 짜임 자체의 보편 규칙이 아니라 이 앱의 민가 팔레트 위계를 판정하는 시각 대조로만 쓴다.

**구현 해석**: `jeongja`·`sesal`은 살 짜임 이름일 뿐 궁궐색을 뜻하지 않는다. `src/builder/palette.js`에서 기와집 변주용 정자살·세살은 갈색 백골 목재와 미색 한지를 사용하며, 궁궐용 빗꽃살의 주칠·뇌록 텍스처를 재사용하지 않는다. 얕은 입체 프레임·문지방·철물과 LOD 경계는 [`exterior-detail.md`](exterior-detail.md)가 소유하고, 궁과 사찰 단청 위계는 [`dancheong.md`](dancheong.md)가 소유한다.

`src/layout/residential-openings.js`는 이 창·문·머름의 공간 어휘를 초가와 기와 ㅡ·ㄱ·ㄷ 평면의 의미 슬롯으로 번역한다. 조절값은 문·창 개수와 각 칸의 사용 가능 폭에 대한 배수 네 개만 소유하며, 남향 주 출입문은 정확히 하나로 정하고 부엌 아궁이 개구는 별도 service opening으로 남긴다. 개수 상한과 폭 배수 범위는 기둥·대청·부엌 개구와 충돌하지 않기 위한 **앱의 칸·충돌 안전 범위**이지, 문헌에서 관찰한 역사적 빈도·전국 분포·보편 비례가 아니다. 현재 단계는 Three 없는 계획 계약만 제공하며 실제 FULL/MID 형상 적용은 후속 렌더 통합에서 검증한다.

`src/layout/choga-shape.js`는 초가 계획·`computeLayout`·FULL 조립이 함께 쓰는 형상 프레임이다. 지원 범위는 정면 3–9칸·측면 1–5칸의 정수, 칸폭 1.4–5m, 기둥 반지름 0.05–0.35m다. 이 범위 밖의 **유한한** 형상값은 실제 생성기와 다른 집으로 조용히 축소하지 않고 `RangeError`로 거절하며, 생략값·비유한값은 계획과 실제 형상 모두 같은 기본 프레임으로 안전하게 복구한다. 기와 형상은 `src/layout/giwa-footprint.js`가 FULL·impostor·fit·개구 계획에 쓰는 단일 정규화 소스다. 주칸 1.8–3.6m, 본채 반폭 2.4–12m·반깊이 1.6–6m, 날개 길이 2.6–12m·폭 1.6–8m, 기둥 반지름 0.05–0.4m로 제한하고 같은 정규화된 footprint·주칸·기둥 반지름을 실제 기와집 조립기도 소비한다. 이 수치도 고증 통계가 아니라 유한 geometry와 편집 안전을 위한 제품 입력 계약이다.

### 2.3 지붕 기와와 장식

- AURI 지붕 해설은 서까래·산자/개판 위에 암키와와 수키와를 잇고 용마루를 마감하는 층위를 설명한다. 현재 FULL 지붕은 이 수키와 골과 마루 실루엣을 절차적으로 요약한다.
- 한국학중앙연구원 「잡상」과 국가유산청 궁궐 지붕 장식 해설은 한국의 잡상을 궁궐 문루·관아 등 위계 높은 건축의 장식으로 설명하고, 국내 사찰에는 잡상 사례가 없다고 정리한다.

**구현 해석**: cheoma의 현행 유형 중 잡상·취두를 받는 유형은 `palace`뿐이다. `roofType: 'paljak'`만으로 장식 위계가 올라가지 않으므로, 사찰 프리셋을 팔작으로 바꿔도 `palace-japsang`과 `palace-chwidu`가 생기지 않는다. 이는 모든 역사적 관아·문루를 궁궐로 분류한다는 뜻이 아니라, 아직 별도 관아 지붕 위계를 갖지 않은 현 생성기의 보수적 경계다.

### 2.4 담장과 마당

담 종류, 몸채 지붕과 담 지붕의 관계, 계층별 경계, 비워 둔 마당 중심과 가장자리 생활물의 근거·알고리즘은 [`village-walls-parcels.md`](village-walls-parcels.md)가 소유한다. 이번 감사에서는 담장을 일괄 추가하거나 마당 중앙에 장식물을 채우지 않았다. 기존 `yard-layout.js`의 장독대·낟가리·빨래줄·텃밭·부속채 footprint와 도로측 대문 계약을 유지한다.

## 4. 자동 계약

- `npm run check:building-clearance`는 두 살림집의 부엌 장면에 `kitchen-opening`, `kitchen-threshold`, `kitchen-door`, `hearth-pot`, `agungiEmber`, `agungiFire`가 한 번씩 있고, 개구가 외벽 밖 독립 매스로 되돌아가지 않는지 검사한다.
- `npm run check:app`은 실제 브라우저에서 민가 정자살·세살의 녹색 우세 픽셀이 사라지고 궁궐 창호색은 남는지, 합성 팔작 사찰에는 궁궐 지붕 장식이 없고 궁에는 있는지 검사한다.
- 같은 앱 게이트는 실제 낙관으로 Reference 모달을 열어 이 감사의 두 선별 출처 묶음, 적용 문구, 원문 링크와 안전한 외부 링크 속성을 확인한다.
- `npm run check:worker`는 이 FULL 건물 지오메트리 변경 뒤에도 sync/Worker/fallback 장면이 바이트 수준에서 일치하는지 검사한다. plan과 seed RNG는 이번 수정에서 바뀌지 않는다.

## 5. 시각 판정 기준

1. 초가와 기와집 측면에서 부뚜막이 집에 덧댄 상자처럼 돌출되지 않아야 한다.
2. 문틀 안쪽의 낮은 솥과 화구는 가까이에서만 읽히고, 부감·MID/FAR 실루엣을 바꾸지 않아야 한다.
3. 밤 불빛은 부엌 개구에 중심을 두며 집 전체를 흰색으로 날리지 않아야 한다.
4. 민가 창호는 궁궐의 주칠·뇌록과 분리되고 백골 목재·한지 몸체와 어울려야 한다.
5. 궁의 잡상 실루엣은 유지하고 사찰 팔작 변형에는 나타나지 않아야 한다.

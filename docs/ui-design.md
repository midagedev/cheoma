# cheoma 웹 UI 설계

목표: 첫 15초 안에 "찍어서 올리고 싶다"를 만든다. 리트윗 유발 3장치 — ① 히어로 조립 오프닝 ② 리롤의 도파민 ③ 낙관 찍힌 엽서 공유. UI 크로마는 최소화하고 씬이 주인공.

## 1. 진입 시퀀스 (히어로)

1. 로드 → 수묵 산수 배경 + 옅은 안개
2. 지형 융기 → 기단·기둥·보·지붕 순 조립 (기존 `src/anim/assembly.js` 재사용, window 훅 経由)
3. 완성 순간 낙관 '처마' 전각이 화면 구석에 쾅 (포스트카드 낙관 재사용) + 먹 타이틀 페이드
4. 스킵: 클릭/스페이스/`?hero=0`
5. (v4 이후) 마을 전체가 안쪽→바깥쪽 스태거로 파도치듯 조립 — 집당 딜레이 0.2~0.4s

## 2. 레이아웃 (크로마)

```
┌────────────────────────────────────────┐
│                            ◔ 환경 다이얼 │  우상: 시간·계절·날씨 (3링 원형 다이얼)
│                                        │
│   (씬 — 풀블리드)              ┆한지 패널┆  집 선택 시에만 슬라이드 인
│                                        │
│ 처                                      │  좌하: 세로쓰기 '처마' + 씨앗번호 한지 라벨
│ 마  #4821                [다시 짓기][엽서][♪] │  우하: 주요 액션 (도장 모양 버튼)
└────────────────────────────────────────┘
```

- 버튼·패널은 한지 텍스처 + 먹선 + 전각 도장 미감. 폰트: 명조 계열(본문), 캘리그래피(타이틀).
- 마우스 없을 때 크로마 자동 페이드 (3s) — 감상 모드.

## 3. 핵심 인터랙션

### 리롤 ("다시 짓기")
- 기존 셔플 훅 + 재조립 애니메이션. 씨앗 번호 갱신, URL `?seed=` 동기화 → 링크 공유 시 같은 마을 재현.

### 집 선택 → 파라미터 패널
- 호버: 은은한 윤곽 하이라이트(림/아웃라인). 클릭: 카메라 포커스 전환(기존 리그 재타깃) + 우측 한지 패널.
- 패널 구성 (먹선 슬라이더·도장 탭):
  - **유형**: 궁 / 절 / 기와집 / 초가 (탭)
  - **칸 들이기 (확장)**: `1R → 2R → 3R` 스테퍼 — 누를 때마다 평면이 一자 → ㄱ자 → ㄷ자로 칸이 늘고, **새 칸만 그 자리에서 조립 애니로 자라남** (이 확장 순간이 클립 감이라 부드럽게)
  - 지붕 물매 / 처마 깊이 / 담장·문 / 마당 소품 밀도 / (궁·절) 단청 강도
- 슬라이더 조작은 디바운스 재생성, 조작 중 실루엣 프리뷰.

### 머지 (합치기)
- 집(또는 채)을 선택하면 인접한 합치기 후보에 은은한 점선 윤곽 → "이웃과 합치기" 도장 버튼 또는 선택 상태에서 후보 클릭.
- 실행 시 두 매스가 서로를 향해 **두부처럼 말랑하게 끌려와** 붙고, 접합부에 새 칸·회첨 지붕이 조립되며 ㄱ/ㄷ자 큰 집으로 재편.
- 단일 필지 단계(P2)에서는 본채+부속채(행각 어휘) 사이 머지로 데모, v4 마을에서 이웃 필지 머지로 확장.
- 역방향(나누기)은 v4 백로그.

### 애니메이션 언어 — "두부 물리" (전 인터랙션 공통)
- 모든 등장·확장·머지 스텝은 **띠용띠용 탄성 이징**: overshoot(back/elastic.out 계열) + 스쿼시&스트레치 (착지 순간 y 눌림·xz 퍼짐 → 복원 출렁 1~2회).
- 기둥은 눌렸다 튀어오르고, 지붕은 안착 때 한 번 출렁, 새 칸은 두부처럼 말랑하게 자리잡음. 스텝 간 스태거로 리듬감.
- 수묵 산수(정적·고요) 위에 두부 물리(귀엽고 통통)의 대비가 이 앱의 시그니처 감성 — Townscaper 계열의 만족감을 조선 미감으로.
- 구현 위치: 코어 src/anim/assembly.js 의 이징을 확장 (UI 레이어에서만 흉내내지 말 것 — 조립·확장·머지 모두 동일 언어 공유).

### 환경 다이얼 (우상)
- 링 1: 시간대 (아침·낮·석양·밤 — 돌리면 해가 실제 이동, 석양 = 골든아워 플래그십 룩)
- 링 2: 계절 (봄·여름·가을·겨울) / 링 3: 날씨 (맑음·비·눈)
- 다이얼 조작 자체가 데모 소재가 되도록 관성·스냅 감 있게.

### 엽서 (포스트카드)
- 현재 앵글 → 기존 postcard 모듈로 낙관+여백 조판 → PNG 다운로드 + Web Share API.
- 엽서 하단에 씨앗 번호·URL 각인 → 엽서 자체가 유입 경로.

## 4. 기술 구조

- **Svelte 5 + Vite**, `app/` 디렉토리 자립(자체 package.json). 코어는 `../src` ES 모듈 직접 import — 코어는 프레임워크 무관 유지(결정 사항).
- 기존 `index.html`+`main.js` 데모 셸은 그대로 보존 (shot 파이프라인·에이전트 검증용). SPA는 별도 엔트리로 병행하다가 배포 시 대표 엔트리 승격.
- 상태: seed·preset·time·season·weather를 URL 쿼리와 양방향 동기화.
- 기존 window 훅·URL 파라미터(조립·히어로·포스트카드·셔플) 재사용 우선, 부족한 훅은 코어에 최소 추가(단, src/ 소유권 비는 시점에).
- v4 마을 모드 통합 지점: `village` 스토어 스텁 (집 목록·선택 인덱스) — 지금은 단일 필지, v4가 채움.

## 4.5 마을 모드 ↔ 집 편집 모드 (P3 핵심 UX)

두 모드의 왕복이 앱의 척추. 판정 기준(사용자 지정): **쾌적한 왕복 · 호버가 쉽게 보이고 쉽게 잡히는 선택 · 선택 시 자연스러운 줌인 진입**.

### 모드 구조
- **마을 뷰**: 부감~사선 궤도(느린 자동 회전), planVillage 결과 전체. 우하 액션이 마을 스코프(다시 짓기=마을 리롤), 패널은 마을 옵션(규모·민촌/여염/반촌·궁·절 다이얼).
- **집 편집 뷰**: 기존 단일 카메라 리그 + 한지 파라미터 패널. 편집 결과는 마을 내 해당 필지에 반영.

### 호버 (마을 뷰)
- 픽킹은 메시가 아니라 **필지 단위 프록시(바운딩)** 레이캐스트 — 지붕·담·마당 어디를 가리켜도 그 집이 잡힘. 작은 초가도 놓치지 않게 프록시를 실제보다 살짝 크게.
- 표시: 먹선 아웃라인 + 집 전체가 아주 살짝 숨쉬듯 밝아짐 + 커서 pointer + 낙관형 미니 라벨(유형·칸수). 60fps 유지(아웃라인은 선택 프록시에만).

### 선택 → 줌인 진입
- 클릭 시 카메라가 그 집으로 **돌리-인 2~2.5초(이즈 인-아웃)**, 목표 프레이밍은 기존 단일 뷰 three-quarter와 동일 → 도착과 동시에 한지 패널 슬라이드 인. 전환 중 DoF가 서서히 조여 주변이 부드럽게 뭉개지면 시선이 집으로 모임.
- 전환 중 클릭 무시(더블 트리거 방지), ESC로 전환 중단 가능.

### 편집 → 마을 복귀
- ESC / 패널 닫기 / 줌아웃 제스처(휠) → 카메라가 마을 부감으로 되돌아감(같은 이징). 방금 편집한 집이 잠깐 은은히 하이라이트되어 "내 집이 저기" 앵커 유지.

### 성능 전제
- 마을 뷰 실시간 60fps: 집 프로토타입 서브지오메트리 InstancedMesh 병합(수십 채→수 드로우), 호버 아웃라인은 프록시 1개 대상. capital 75호가 기준선.

## 5. 단계

- **P1 (지금)**: 스캐폴드 + 히어로 + 리롤 + 환경 다이얼 + 엽서 (단일 건물 씬)
- **P2**: 집 선택·파라미터 패널·칸 확장 스테퍼
- **P3 (v4 후)**: 마을 모드(4.5절) — 마을 스태거 조립·집 순회 카메라·1인칭

# 조선 궁궐 배치 문법 — 절차 생성 규칙 (태스크 #88 리서치)

조사일: 2026-07-18. 목적: 마을 생성기의 궁을 "담 하나에 정전 한 채"에서 **경복궁·창덕궁처럼 여러 전각군(일곽)이 행각으로 담을 공유하는 궁역**으로 격상하기 위한 배치 규칙. 구현 에이전트가 그대로 코딩할 수 있게 의사코드·파라미터·앱 환산 수치로 정리했다. 역사 서술이 아니라 **공간 패턴·비율·수치** 중심.

## 0. 핵심 명제 (한 줄 요약)

> **조선 궁궐은 "필지의 집합"이 아니라 "행각(行閣)으로 둘러싸인 마당(일곽) 여러 개를 축선 위에 꿴 것"이다. 인접한 일곽은 독립 담이 아니라 하나의 행각을 경계로 공유한다. 담 = 행각. 정전→편전→침전으로 갈수록 마당이 작아지고 격식이 낮아진다.**

현재 코드(`buildPalaceCore` → `buildParcel({style:'palace'})`)는 이 중 **딱 한 일곽(정전 일곽)** 만 만든다. 격상의 본질은 (a) 일곽을 3~4개로 늘리고, (b) 축선에 남→북으로 꿰고, (c) 인접 일곽이 행각을 공유하게 하고, (d) 동/서에 부속 영역(동궁·궐내각사)을 붙이는 것이다. 신규 빌더는 최소화하고 기존 `buildCorridor`(행각)·`buildBuilding`(전각)·`buildGate`(문)·`buildBridge`(금천교)·`buildPodium`(월대)를 조합한다.

## 1. 공간 위계 — 영역(일곽) 구분과 상대 위치

궁역은 기능·격식이 다른 **영역(일곽, 一廓)** 들의 집합이다. 각 영역은 "전각 1채(+부속) + 그 앞 마당 + 마당을 두르는 행각 + 남측 정문"의 한 세트. 축선(남북) 기준 위치와 기능:

| 영역 | 대표 전각 | 축선상 위치 | 기능 | 마당 격식 | 상대 크기 |
|---|---|---|---|---|---|
| **외전 – 정전(正殿)** | 근정전 | 축선 남측(진입 후 첫 대형 일곽) | 즉위식·조회·사신맞이. 궁의 얼굴 | 최대. 조정(朝廷)=품계석+박석 | 100% (기준) |
| **외전 – 편전(便殿)** | 사정전 | 정전 **바로 북** | 왕의 일상 집무·경연 | 중. 정전보다 작음 | 폭 0.75, 깊이 0.7 |
| **내전 – 침전(寢殿)** | 강녕전 | 편전 **바로 북** | 왕의 침소·생활 | 중소. 마당 아담·부속 전각 4채 일곽 | 폭 0.8, 깊이 0.75 |
| **내전 – 중궁전** | 교태전 | 침전 **바로 북**(축선 종결) | 왕비 처소. 뒤에 후원(아미산) | 소. 가장 사적 | 폭 0.7, 깊이 0.65 |
| **동궁(東宮)** | 자선당·비현각 | 정전~편전 축의 **동측(+동)** | 세자 처소 | 별도 소일곽 1~2 | 폭 0.55 (측면 블록) |
| **궐내각사(闕內各司)** | 수정전 등 | 정전~편전 축의 **서측(-동)** | 근무 관청. 여러 작은 채가 밀집 | 격자형 작은 마당들 | 폭 0.6 (측면 블록) |
| **후원(後苑)** | 아미산·향원정 | 침전군 **북측 여백** | 정원. 건물 희소, 지형·수목 | 비건축(연못·정자·화계) | 여백 |
| **궐외각사** | 육조 | 궁 **정문 밖 남측** | 의정부·육조 등. 이미 도성 `planParcels`/`govCore` 영역 | (도성 골격 소관) | 궁역 밖 |

**영역 그래프(무엇이 무엇의 어느 쪽인가)** — 절차 생성의 뼈대:

```
                      [후원/아미산]   (N, 여백·비건축)
                            │
                      [중궁전 교태전]
                            │  (행각 공유)
                      [침전 강녕전] ──(+동 부속 4채)
                            │  (행각 공유)
   [궐내각사]────(-동)── [편전 사정전] ──(+동)──── [동궁]
   (서측 블록)               │  (행각 공유)          (동측 블록)
                      [정전 근정전]  ← 조정(품계석 마당)
                            │
                      [정문: 근정문]
                            │
                    ── 금천/금천교 ──   (정문과 중문 사이)
                            │
                      [중문: 흥례문]
                            │
                      [정문/궁성 남문: 광화문]  (S, 진입)
```

좌표 규약(앱): **남=+z(진입·정문), 북=-z(배산·후원)**. 전각은 모두 +z(남) 정면. 동궁=+x축 부속, 궐내각사=-x축 부속(좌묘우사와 무관한 궁 내부 배치이나, 실제 경복궁도 동궁이 동쪽이라 +x=동 규약과 일치).

## 2. 축선과 문 시퀀스 (경복궁 = 정형 축)

경복궁은 평지형 정궁이라 **엄격한 직선 남북축**에 문·마당·전각을 대칭 반복한다. 축선 시퀀스(남→북), 각 사이는 "문 통과 → 마당 → 다음 문/전각"의 리듬:

```
광화문(궁성 남문, 3홍예)
  → [진입 마당]
興禮門 흥례문(중문, 3칸)
  → [금천 마당] 영제교(금천교)로 명당수 건넘   ← 금천은 정문과 중문 사이
勤政門 근정문(정전 정문, 3칸)
  → [조정 = 근정전 마당] 품계석·박석
勤政殿 근정전(정전, 2중 월대)          ← 외전 시작
  → 思政門 사정문
思政殿 사정전(편전, +동서 만춘전·천추전)
  → 향오문 등
康寧殿 강녕전(침전, +경성·연생·연길·응지 4전)  ← 내전 시작
  → 양의문
交泰殿 교태전(중궁전)
  → 아미산(후원, 축선 종결)
```

**리듬 규칙**: 문을 지날 때마다 마당이 하나 나오고, 각 마당의 북단에 전각이 앉는다. 마당 크기는 남(정전)에서 북(중궁전)으로 갈수록 **단조 감소**. 이것이 "속(俗)→성(聖)/공(公)→사(私)"의 위계 상승 축(산사 가람의 문 시퀀스와 같은 원리, joseon-city §5·§7-G).

**대칭**: 각 전각은 축선 좌우 대칭. 행각·문·계단이 중심선 기준 미러. 절차 생성 시 각 일곽을 **축선(x=0) 대칭으로** 배치하면 궁다운 위엄이 산다(마을 필지의 부정형과 정반대 — 궁은 결정론·대칭이 핵심).

출처: [sillokwiki 경복궁](https://dh.aks.ac.kr/sillokwiki/index.php/%EA%B2%BD%EB%B3%B5%EA%B6%81(%E6%99%AF%E7%A6%8F%E5%AE%AE)), [우리역사넷 도성과 궁궐](https://contents.history.go.kr/mobile/nh/view.do?levelId=nh_035_0040_0020_0050_0020), [경복궁전도(refs)], [영제교 치수 – en.wikipedia Gyeongbokgung](https://en.wikipedia.org/wiki/Gyeongbokgung).

## 3. 행각·담 공유 구조 (구현의 심장)

### 3-1. 원리

- **행각(行閣)/회랑(廻廊)**: 폭 1~2칸의 좁고 긴 건물이 마당을 **ㅁ자로 두른다**. 바깥면=회벽(담 역할), 안마당면=개방 기둥열. 즉 **행각의 바깥벽이 곧 궁의 내부 경계담**이다 — 독립 담장을 따로 세우지 않는다.
- **담 공유**: 정전 일곽의 **북측 행각**과 편전 일곽의 **남측 행각**은 축선상 맞닿는다. 실제 궁에서는 이 경계를 **한 겹의 행각(또는 전각+행각)이 겸한다** → 두 마당이 하나의 행각선을 공유. 절차 생성에선 인접 일곽 사이에 행각을 **한 번만** 세우고, 그 선에 다음 마당으로 통하는 **문간(중문)** 을 뚫는다.
- **복랑(複廊) vs 단랑(單廊)**: 격식 높은 정전 일곽은 **복랑**(2줄 기둥, 깊이 큼), 편전·침전은 **단랑**(1줄). 앱에선 `buildCorridor`의 `depth` 파라미터로 표현(정전 depth≈3.0 복랑감, 그 외 2.4 단랑).

### 3-2. `buildCorridor` 재사용 규칙 (신규 빌더 불필요)

`src/layout/corridor.js`의 `buildCorridor({points, closed, mats, seed, depth, bayLen, colH, openEvery, innerSide})` 가 이미 폴리라인 기반 행각을 세운다. 폐곡선(`closed:true`)이면 ㅁ자 회랑. 이걸 일곽마다 호출한다.

```
function buildCourtyardEnclosure(cx, cz, W, D, opts):
  # 마당 중심 (cx,cz), 내부 마당 폭·깊이 W×D. 행각은 이 사각을 두름.
  hw = W/2; hd = D/2
  # 남측 변에는 정문 개구부, 북측 변에는 다음 일곽으로 가는 문간 개구부
  corners = [SW, SE, NE, NW]   # CCW
  buildCorridor({
    points: corners, closed:true, mats,
    depth: opts.rank==='jeongjeon' ? 3.0 : 2.4,   # 복랑/단랑
    colH:  opts.rank==='jeongjeon' ? 3.2 : 2.7,   # 정전 행각이 더 높음
    openEvery: 0,                                  # 개구는 문으로 처리
    innerSide: +1,                                 # CCW 폐곡선은 +1이 마당쪽 개방
  })
  # 남변 중앙에 정문(buildGate), 북변 중앙에 문간(작은 gate 또는 행각 끊고 문지방)
```

### 3-3. 공유 경계 처리 — 두 방식

1. **간단(권장 1차)**: 각 일곽을 **독립 폐곡선 행각**으로 만들되, 남북으로 **행각 두께만큼 겹쳐 배치**해서 시각적으로 한 벽처럼 붙게 한다. 겹치는 구간은 z-fighting 방지 위해 북측 일곽의 남변 행각을 생략(아래 §3-4 개구 규칙). 구현 난이도 최소, 회귀 위험 낮음.
2. **정합(2차 폴리시)**: 인접 두 일곽 사이 경계 행각을 **한 번만** 생성하고 양쪽이 참조. 폴리라인을 이어 그리면 되지만 개구부·문간 처리가 복잡. 1차 안정화 후 고려.

### 3-4. 문간·모서리

- **남측 정문 개구**: 각 일곽 남변 행각 중앙을 `gateW` 만큼 비우고 `buildGate` 배치(정전=`soseuldaemun` 3칸급, 편전·침전=`iljakmun` 1칸급).
- **북측 통행 개구**: 다음 일곽으로 통하는 문. 정전↔편전은 격식 문, 침전 사이는 간소.
- **모서리**: `buildCorridor`가 폐곡선 꺾임에 모서리 기둥을 이미 세운다. 행각이 직교로 만나는 궁의 특성과 일치.
- **행각 두께(깊이)**: 실측 행각은 대략 1칸≈2.5~3.0m 폭. 앱 `depth` 2.4(단랑)~3.0(복랑) 유지. 행각 지붕은 `buildCorridor` 내부에서 맞배(`buildTileGableRoof`)로 처리됨.

## 4. 마당(조정) 문법

### 4-1. 정전 마당 = 조정(朝廷)

- **박석(薄石)**: 다듬지 않은 넓적한 화강석 판석을 마당 전면에 깖. 표면 거칢 = 걸음 조심의 상징. 앱: 마당 바닥을 밝은 회백색 거친 재질(현 `yard` 흙색 대신 `stone` 계열 평면)로. 살짝 요철/불규칙 UV면 더 좋음(비용 0에 가깝게 색만 바꿔도 충분).
- **품계석(品階石)**: 마당 좌우에 **동·서 각 12개씩**(정1품~종9품, 1~3품은 정/종 구분해 6쌍, 나머지 6개) 작은 돌기둥을 두 줄로 세움. 앱: 축선 좌우 대칭으로 z를 따라 12개씩 작은 실린더/박스(props 재사용 가능). 정전 일곽 전용.
- **어도(御道)**: 정문→전각 계단까지 마당 중앙을 관통하는 약간 높은 3열 답도. 현 `buildParcel`의 디딤돌 길을 **넓은 3열 포석 어도**로 승격(정전 일곽 한정). 중앙열에 답도석(봉황/용 조각은 생략 가능).

### 4-2. 마당 비례

- 정전 조정은 **세로로 긴** 마당(남북 > 동서). 실측 근정전 조정 ≈ 남북 100m × 동서 60~65m → 비율 약 **1.5:1(깊이:폭)**. 앱에서도 정전 마당은 세로로 길게.
- 편전·침전 마당은 정전보다 작고 정방형에 가까움(1.1~1.2:1).
- **행각 안쪽 순 마당(전각 제외)** 폭은 전각 정면폭의 약 2~2.5배, 깊이는 전각~정문 사이 3~4칸 여유.

## 5. 금천(禁川)과 금천교

- **규칙**: 궁 진입부, **정문(광화문)과 중문(흥례문) 사이**에 명당수(금천)가 동서로 흐르고, 그 위에 **금천교**(경복궁=영제교)가 남북으로 놓여 축선을 잇는다. "궁에 들 때 물을 건너 부정을 씻는다"는 의례 장치.
- **치수**: 영제교 실측 약 **폭 10m × 길이 13m**. 앱에선 홍예교(`buildBridge({type:'arch'})`)를 축선 위에 1개. 폭은 어도 폭에 맞추고(≈6~8m), 개천은 얕은 도랑+물 평면.
- **변주(창덕궁)**: 창덕궁 금천교는 명당수가 **북→남**으로 흘러 다리가 **동서 방향**으로 놓이고 진입이 꺾인다(§7). 경복궁형(정형)은 남북 다리, 창덕궁형(비정형)은 동서 다리 + 진입 굴절.

## 6. 수치 비율과 앱 환산 (가장 중요)

### 6-1. 실측 기준

- 경복궁 전체: 창건 약 **343,888㎡**, 대략 **동서 ~500m × 남북 ~700m** 급, 총 7,000여 칸·전각 수백 채. → **1:1로 앱에 넣으면 R500 도시를 통째로 잡아먹는다. 반드시 축소·발췌한다.**
- 근정전 조정 일곽: 남북 ~100m × 동서 ~65m, 44칸 행각.
- 영제교: 10×13m. 품계석: 동서 각 12.

### 6-2. 앱 스케일 원칙

**전각·문·행각은 현 1:1 유지(집 15m 옆에서 어색하지 않아야 함), 축소는 "일곽 개수·마당 크기·전각 수"에서 한다.** 즉 경복궁 700m 축선을 통째 재현하지 않고 **대표 4일곽으로 발췌**한다.

기준 모듈: 앱 정전(`buildBuilding` korea 프리셋) footprint ≈ **정면 15.3m × 측면 8.7m**(5×3칸). 이걸 1 모듈로 삼으면:

| 요소 | 순 마당 W×D | 행각 포함 외곽 W×D | 비고 |
|---|---|---|---|
| 정전 일곽 | 28×34 | **34×40** | 조정(품계석·박석·어도). 세로 긴 비례 |
| 편전 일곽 | 22×22 | 28×28 | 만춘/천추 생략 or 동측 소전 1채 |
| 침전 일곽 | 24×22 | 30×28 | 부속 4전은 마당 모서리 소형 채로 암시 |
| 중궁전 일곽 | 18×18 | 24×24 | 축선 종결. 북에 후원 여백 |
| 동궁(동측 블록) | — | 22×26 | 정전~편전 축 동측, 소일곽 1 |
| 궐내각사(서측 블록) | — | 22×30 | 작은 채 3~4 밀집(격자 마당) |
| 진입부(금천~중문) | — | 34×26 | 금천교+흥례문 마당 |

**축선 스택(남→북) 깊이 합**: 진입 26 + 정전 40 + 편전 28 + 침전 28 + 중궁 24 + 후원 여백 ~14 ≈ **160m 깊이**. **폭**: 축선 코어 34 + 동궁 26 + 궐내각사 30 = **~90m**(측면 블록은 정전~편전 구간에만 붙으므로 최대폭 구간 한정).

### 6-3. 궁역 크기 권고 (현 66×58 대비)

| 티어 | 현재 | 권고 | 구성 | 근거 |
|---|---|---|---|---|
| **hanyang 플래그십** | 66×58 | **96 × 150** | 4일곽 축선 + 동궁 + 궐내각사 + 금천교 | 위 스택(90×160)을 후원 여백 축약해 수용. 현 대비 폭 ~1.45×, 깊이 ~2.6×, 면적 ~3.8× |
| **hanyang 최소(예산 부족 시)** | 66×58 | **70 × 108** | 3일곽(정전+편전+침전) + 금천교, 측면 블록 생략 | 축선 코어만. 담 공유 문법은 그대로 시연 |
| **capital 축소판** | 46×40 | **60 × 90** | 3일곽 축선(측면 블록 생략) or 창덕궁형 2일곽+지형 굴절 | 회귀 안전 위해 capital 은 보수적 확장 |

> **권고 요지**: 현 66×58은 **정전 일곽 하나(34×40)** 를 겨우 담는 크기 — 사용자 지적("단일 필지")의 정확한 원인. 다중 일곽·담 공유를 시연하려면 **깊이를 최소 2배(≥108), 폭을 1.4배(≥90)** 로 늘려야 한다. 플래그십 96×150 을 권장하되, R500 도시·성곽·시전 예산과 충돌하면 최소 티어 70×108 로 시작해 측면 블록을 후속 추가.

궁역 확장은 `plan.js` L68 (`pw/pd`)과 여기서 파생되는 `blockers` 폴리곤, `citywall`·`planParcels` 간섭만 조정하면 된다(궁역이 커지면 주변 필지·시전이 밀려남 — 도로/필지 재계산 필요).

### 6-4. 드로우콜/성능 예산

- 궁은 도시당 **1개뿐인 히어로 피처** → 마을 수천 채 대비 절대 메시 수 여유 있음. 그래도 `buildCorridor`는 세그먼트별 개별 그룹(인스턴싱 아님)이라 4일곽 ×4변 = 16+ 행각 런 + 모서리 기둥 다수 → 수백 메시.
- 규칙: (a) 행각은 **팔레트 공유 재질**(현 corridor는 이미 `mats` 공유)로 드로우콜 억제, (b) 전각은 palace 프리셋 재사용(신규 재질 금지), (c) 품계석·초석 같은 반복 소품은 **InstancedMesh**(마을 `instancing.js` 어휘 재사용) 권장, (d) 원경 LOD 필요 시 hanyang far-chunk 임포스터 경로에 궁역도 편입 검토.
- **신규 빌더 0 목표**: 정전/편전/침전=`buildBuilding`(korea·프리셋 변형), 담·경계=`buildCorridor`, 문=`buildGate`, 금천교=`buildBridge('arch')`, 월대=`buildPodium`, 마당 소품=props/instancing. 새 파일이 필요하다면 이 조각들을 **조립하는 오케스트레이터**(예: `buildPalaceCore` 확장 또는 `src/village/palace.js` 신설) 하나면 충분하다.

## 7. 창경궁·창덕궁 변주 (절차 생성 변주 축)

정형(경복궁)은 하나의 극이고, 지형 순응형(창덕궁)이 반대 극이다. 이 둘을 **`layout` 파라미터('axial'|'terrain'|'east')** 로 스위칭하면 궁 생성에 다양성이 생긴다.

| 축 | 경복궁형 `axial` | 창덕궁형 `terrain` | 창경궁형 `east` |
|---|---|---|---|
| 지형 | 평지 | 산기슭(응봉 자락) | 완경사 |
| 축선 | **엄격한 직선 남북축**, 좌우 대칭 | **없음**. 정문 남향이나 진입 후 동향으로 꺾고 전각군이 골짜기마다 흩어짐 | 전각이 **동향**(정문 홍화문·정전 명정전이 동쪽 봄) |
| 정문→정전 | 일직선 | 돈화문(남향)→금천교(동향 진입)→진선문→인정문→인정전(다시 북) = **L자 굴절** | 홍화문(동)→옥천교→명정문→명정전 = 짧은 동향축 |
| 금천교 | 명당수 동서류 위 **남북 다리** | 명당수 남북류 위 **동서 다리** | 옥천교(동서 진입) |
| 일곽 배치 | 축선에 **일렬로 스택**, 대칭 | 지형 단(段)마다 **분산·비대칭**, 향(向) 제각각 | 정전 동향, 침전은 남향 등 **향 혼재** |
| 절차 규칙 | 일곽 중심 x=0 고정, z만 증가 | 일곽마다 지형 고도 따라 위치·회전 지터, 진입축 1회 90° 굴절 | 정전 일곽만 동향(rotateY+90°), 나머지 남향 |

**구현 제안**: 1차는 `axial`(경복궁형)만 견고히. 변주는 후속. `terrain` 형은 마을 지형(`site.heightAt`)과 결합해 일곽마다 성토 패드 고도를 달리하고 진입 동선을 꺾으면 창덕궁 특유의 "산속 궁궐"이 난다(산사 가람 §7-G 규칙과 공유). `east` 형은 정전 일곽 회전만으로 저비용 변주.

출처: [창덕궁 – ko.wikipedia](https://ko.wikipedia.org/wiki/%EC%B0%BD%EB%8D%95%EA%B6%81), [창덕궁 금천교 동서 방향 – encykorea](https://encykorea.aks.ac.kr/Article/E0055380), [동궐도(refs) – 창덕궁·창경궁 지형 순응 조감].

## 8. 절차 생성 파라미터·의사코드 (구현 직결)

### 8-1. 파라미터 스키마

```js
palaceSpec = {
  layout: 'axial',              // 'axial'|'terrain'|'east'
  regionW: 96, regionD: 150,    // 궁역 외곽 (§6-3 티어)
  center: {x, z}, frontDir,     // 진입 방향 (남=+z)
  areas: [                       // 축선 남→북 순서. 각 = 일곽 스펙
    { role:'jeongjeon', W:34, D:40, corridorDepth:3.0, colH:3.2,
      gate:'soseuldaemun', hall:'korea', podium:'double', court:'jojeong' },
    { role:'pyeonjeon', W:28, D:28, corridorDepth:2.4, colH:2.7,
      gate:'iljakmun', hall:'korea-small', podium:'single' },
    { role:'chimjeon',  W:30, D:28, corridorDepth:2.4, colH:2.6,
      gate:'iljakmun', hall:'korea-res', podium:'single', satellites:4 },
    { role:'junggung',  W:24, D:24, corridorDepth:2.4, colH:2.5,
      gate:'iljakmun', hall:'korea-res', podium:'single', backGarden:true },
  ],
  flanks: [                      // 축선 측면 부속 (정전~편전 구간)
    { role:'donggung',   side:+1, W:22, D:26, attachTo:'pyeonjeon' },
    { role:'gwolnaegaksa', side:-1, W:22, D:30, attachTo:'pyeonjeon', subCells:4 },
  ],
  stream: { at:'entry', bridge:'arch' },   // 금천교: 정문·중문 사이
  seed,
}
```

### 8-2. 배치 알고리즘

```
function buildPalace(spec, site):
  cursorZ = center.z + regionD/2         // 남단(진입)에서 시작
  # 1) 진입부: 궁성 남문 + 금천교 마당 + 중문
  place southGate at cursorZ; cursorZ -= entryCourtD
  place archBridge (buildBridge 'arch') over shallow stream; place midGate
  cursorZ -= midCourtD
  # 2) 축선 일곽 스택 (남→북), 담 공유
  for i, area in areas:
      areaCz = cursorZ - area.D/2
      pad = buildPadRect(site, center.x, areaCz, area.W, area.D)   # 지형 성토
      # 행각 폐곡선 (담 = 행각). 남변에 정문 개구, 북변에 통행 개구
      buildCourtyardEnclosure(center.x, areaCz, area.W - 2*corridorDepth, area.D - 2*corridorDepth, area)
      buildGate(area.gate) at south-mid opening
      # 전각 (마당 북측, 남향). 정전은 2중 월대(buildPodium 'double')
      hall = buildBuilding(preset(area.hall))
      place hall at (center.x, north-third of court), facing +z, on buildPodium(area.podium)
      if area.court == 'jojeong': layJojeong(품계석 12쌍 + 박석 + 어도)   # 정전만
      if area.satellites: place N small halls at court corners
      # 인접 일곽과 담 공유: 다음 일곽 남변 행각을 이 북변과 겹치게 (§3-3 방식1)
      cursorZ -= area.D - corridorDepth   # 행각 두께만큼 겹침
  # 3) 측면 부속 (동궁·궐내각사): attachTo 일곽의 x측면에 소일곽
  for flank in flanks:
      fx = center.x + flank.side * (coreHalfW + flank.W/2)
      fz = z-of(area named flank.attachTo)
      buildCourtyardEnclosure(fx, fz, ...) ; fill with small halls (subCells 격자)
  # 4) 후원 여백: 최북단 일곽 뒤는 비건축(수목·화계·정자 props)
  # 5) 궁성 담: 전체 regionW×regionD 둘레에 성벽/궁장 (기존 담 어휘 or citywall 축소)
```

### 8-3. 대칭·결정론 규칙

- 모든 일곽은 축선(center.x) 기준 **좌우 대칭** 배치. `axial` 에선 회전 지터 0.
- `seed` 로 결정론 유지(마을 규약과 동일). 같은 seed → 같은 궁.
- 지형: 각 일곽 아래 `buildPadRect` 성토 패드(마을 `populate.js` 어휘 재사용)로 관통·부양 방지. `terrain` 변주에선 패드 고도를 일곽마다 계단식으로.

## 11. 한 줄 결론

궁 격상 = **`buildPalaceCore`를 "정전 일곽 1개"에서 "행각으로 담을 공유하는 3~4일곽 축선 스택 + 동궁·궐내각사 측면 블록 + 금천교"로 확장하는 오케스트레이션**. 신규 빌더 없이 `buildCorridor`(담)·`buildBuilding`(전각)·`buildGate`(문)·`buildBridge`(금천교) 조립. 궁역은 hanyang 66×58 → **96×150(플래그십)** 또는 **70×108(최소)**, capital 46×40 → **60×90** 로 확장 필요.
